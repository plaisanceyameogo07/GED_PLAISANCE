================================================================================
  ⚡ DÉMARRAGE EN 5 MINUTES
================================================================================

1. LANCER LE SERVEUR
   ─────────────────
   Ouvrez un terminal et tapez :
   
   cd ged-serveur
   node server.js
   
   Vous verrez :
   ✅ Serveur démarré sur le port 3000
   
   ⚠️  NE FERMEZ PAS CE TERMINAL !

2. OUVRIR L'APPLICATION
   ────────────────────
   Ouvrez un navigateur (Chrome, Firefox, Safari, Edge) et allez à :
   
   http://localhost:3000/ged-serveur/GED_Plaisance_v6.html
   
   Ou via fichier :
   file:///C:/Users/SONIA/Desktop/PLAISANCE DOC/PERSONNEL/MON PROJET/GED PLAISANCE/ged-serveur/GED_Plaisance_v6.html

3. VÉRIFIER QUE ÇA MARCHE
   ─────────────────────
   Appuyez sur F12 → Console
   
   Vous devriez voir après 2 secondes :
   ✓ 📥 Démarrage pré-cache des fichiers...
   ✓ ✅ Pré-cache terminé : X fichiers pré-cachés
   
   Si vous voyez ça → C'EST BON ! ✅

4. AJOUTER UN FICHIER
   ──────────────────
   Cliquez sur "➕ Ajouter fichier"
   Choisissez un fichier (photo, PDF, Word, etc.)
   
   Le fichier s'ajoute à la GED

5. OUVRIR UN AUTRE NAVIGATEUR
   ──────────────────────────
   Firefox (si vous aviez Chrome) ou Chrome Incognito
   
   http://localhost:3000/ged-serveur/GED_Plaisance_v6.html
   
   Le fichier que vous avez ajouté APPARAÎT ! 🎉
   
   → SYNCHRONISATION FONCTIONNE !

================================================================================
  ❌ ÇA NE MARCHE PAS ?
================================================================================

ERREUR : "Impossible de se connecter"

   → Le serveur n'est pas lancé ?
     Ouvrez un AUTRE terminal et tapez :
     cd ged-serveur
     node server.js

ERREUR : "localhost:3000 refused"

   → Port 3000 bloqué par le firewall ?
     Autorisez Node.js dans Windows Defender
     OU redémarrez le serveur

ERREUR : "Console dit 'Cannot read property transaction'"

   → Normal ! IndexedDB se charge. Attendez 3 secondes et rafraîchissez (F5)

ERREUR : "Console dit 'fetch failed'"

   → Vérifier que SERVER_URL dans le HTML correspond à l'adresse du serveur
     (ligne ~17 du fichier GED_Plaisance_v6.html)

→ AIDE COMPLÈTE : Voir GUIDE_CONFIGURATION_TESTS.md section "Troubleshooting"
